# 임장 기록 노션 자동 업로드 스킬

사용자가 임장(아파트 방문) 후기 텍스트를 채팅으로 입력하면, 에이전트가 아래 워크플로우를 수행합니다.

## 🔑 사전 준비

1. `.env` 파일에서 인증 정보를 읽습니다:
   - `NOTION_API_KEY`: 노션 Integration 토큰
   - `NOTION_DATABASE_ID`: 대상 노션 데이터베이스 ID
2. `template.md`: 빈 입력 양식 템플릿 (직접 수정 금지, 원본 유지)
3. `upload_to_notion.py`: 마크다운 파일을 파싱하여 노션 API로 업로드하는 스크립트

## 🔄 워크플로우 (전체 흐름)

### Step 1. 사용자 입력 수신

사용자가 텍스트로 임장 후기를 주고, **임장 사진 파일**을 함께 전달할 수 있습니다.
에이전트는 텍스트에서 정보를 추출하고, 제공된 사진 파일들을 안전하게 보관해야 합니다.

**필수 추출 항목:**
- 아파트명 (필수)
- 평형 (선택)
- 방문일 (선택, 없으면 오늘 날짜 사용)

**현장 실사 점수 (1~5점):**
- 현장교통, 현장학군, 현장상권, 현장환경, 현장단지

#### 🚇 현장교통 자동 판단 기준 (에이전트가 네이버 부동산 데이터로 판단)
가장 가까운 지하철역까지 도보 시간 기준:
| 도보 시간 | 점수 |
|---|---|
| 5분 이내 | 5점 |
| 8분 이내 | 4점 |
| 10분 이내 | 3점 |
| 15분 이내 | 2점 |
| 20분 이상 | 1점 |

#### 🏫 현장학군 자동 판단 기준 (에이전트가 네이버 부동산 데이터로 판단)
배정 초등학교까지 도보 시간 기준 (가장 가까운 학교):
| 도보 시간 | 점수 | 비고 |
|---|---|---|
| 단지 옆 (1분 이내) | 5점 | 초품아 (초등학교를 품은 아파트) |
| 3분 이내 | 4점 | |
| 5분 이내 | 3점 | |
| 7분 이내 | 2점 | |
| 10분 이상 | 1점 | |


> **현장상권, 현장환경, 현장단지**는 사용자가 직접 1~5점을 부여합니다.
> 사용자가 점수를 주지 않으면 반드시 물어봅니다.

**현장 메모:**
- 오감 체감 (소음, 냄새, 경사, 관리, 보행 안전, 단지 느낌)
- 일조량 데이터 (동/층별)
- 추가 의견

### Step 2. 네이버 부동산 정보 조회 (자동)

아파트명이 확인되면 에이전트는 사용자에게 네이버 부동산 단지 URL을 입력받아 정보를 가져옵니다:

#### 2-1. 기본 정보 (■ 0) 채우기
1. 사용자에게 다음과 같이 안내하며 **네이버 부동산 단지 URL**을 요청합니다:
   > "네이버 부동산(https://fin.land.naver.com)에서 해당 아파트를 검색한 뒤, 단지 페이지 URL을 붙여넣어 주세요.
   > 예시: `https://fin.land.naver.com/complexes/8083?tab=article`"
2. 사용자가 URL을 입력하면, `browser_subagent`로 **해당 URL에 바로 접속**합니다 (검색 단계 생략).
3. **호가 정보 수집:**
   - 매물 탭에서 평형별로 필터링하여
   - 현재 게시된 매물의 **최저가 ~ 최고가** 범위를 추출
   - 예: `매매 14.8억 ~ 15.5억`

#### 2-2. 사전 조사 (■ 1) 채우기
1. 단지정보 탭(`?tab=complexInfo`)으로 이동하여 스크래핑:
   - **교통 노드**: 가장 가까운 지하철역, 출구, 소요 시간
   - **단지 스펙**: 세대수 / 동수 / 사용승인일 (연차 계산)
   - **인프라**: 주변 편의시설 (백화점, 마트 등)
   - **학군 정보**: 배정 초등학교 / 중학교
2. 네이버 부동산에서 부족한 정보는 `search_web`으로 보완

#### 2-3. 한줄 요약
- 수집된 모든 정보(시세, 사전 조사, 현장 점수)를 종합하여 에이전트가 **직접 작성**

### Step 3. 누락 정보 확인 및 질문

파싱 완료 후, 아래 항목 중 누락된 것이 있으면 사용자에게 질문합니다:

**반드시 확인할 항목:**
- [ ] 현장 5개 점수 (현장교통/학군/상권/환경/단지) — 1~5점
- [ ] 오감 체감 항목 (소음, 냄새, 경사, 관리, 보행, 단지) 중 하나라도 빠진 것

**질문 예시:**
> "현장교통, 현장학군 점수가 빠져있어요. 각각 1~5점으로 평가해주세요.  
> 또한, 소음이나 냄새에 대한 체감도 알려주시면 좋겠어요."

### Step 4. 마크다운 파일 생성 및 사진 자동화

⚠️ **template.md는 절대 수정하지 않습니다.** 항상 새 파일을 생성합니다.

#### 4-1. 사진 처리 및 GitHub 업로드
사용자가 사진 파일을 제공한 경우, 해당 사진들을 GitHub에 업로드하여 외부 참조 가능한 URL을 생성합니다.

1. **폴더 생성**: `c:\develop\notion-housr\photos\[ApartmentName_English]_[Date]\` 폴더를 생성합니다. 
   - **중요**: URL 인코딩 이슈 및 한글 오타 방지를 위해 폴더명은 반드시 영어를 사용합니다.
2. **사진 저장**: 사용자가 보낸 이미지 파일들을 해당 폴더에 저장합니다.
   - **중요**: 이미지 파일명 역시 반드시 영어를 사용합니다. (예: `living_room_01.jpg`, `complex_gate.jpg` 등)
3. **Git 업로드**:
   - `git add photos/`
   - `git commit -m "Add photos for [ApartmentName] [Date]"`
   - `git push`
4. **URL 생성**: 업로드된 각 사진에 대해 GitHub 이미지 URL을 생성합니다.
   - 형식: `https://github.com/devsmilekang/notion-house/blob/main/photos/[ApartmentName_English]_[Date]/[English_FileName]?raw=true`
   - 저장소 정보: `devsmilekang/notion-house`, 브랜치: `main`

#### 4-2. 리포트 파일 생성
1. `template.md`를 읽어서 내용을 복사
2. 수집한 데이터(기본 정보, 시세, 점수 등)로 각 필드를 채움
3. **사진 섹션 채우기**: 생성된 GitHub Raw URL들을 `## ■ 4. 임장 사진 아카이브` 섹션에 한 줄에 하나씩 적습니다.
4. 새 파일명 규칙: `임장기록_[아파트명]_[날짜].md`
5. `c:\develop\notion-housr\records\` 폴더에 저장 (없으면 생성)

### Step 5. 노션 업로드

1. 생성된 마크다운 파일 경로를 인자로 `upload_to_notion.py` 실행:
   ```
   python upload_to_notion.py records/임장기록_롯데캐슬클라시아_20260228.md
   ```
2. 스크립트가 리포트 파일의 GitHub Raw URL을 읽어 노션의 `image` 블록(external)으로 자동 업로드합니다.
3. 업로드 시 `input()` 확인 프롬프트가 나오므로, 파이프로 `y`를 전달:
   ```
   echo y | python upload_to_notion.py records/임장기록_...md
   ```

### Step 6. 결과 보고

성공 시:
> "✅ 성공적으로 노션에 임장 기록을 업로드했습니다! 🎉  
> 📎 페이지 URL: https://www.notion.so/xxxx"

실패 시:
- 에러 메시지를 분석하여 원인 파악
- 노션 데이터베이스 스키마 불일치인 경우, `check_db.py`로 스키마 재확인

## 📋 노션 데이터베이스 스키마

| 속성명 | 타입 | 설명 |
|---|---|---|
| `아파트명` | title | 아파트 이름 (필수) |
| `평형` | select | 평형 (예: 24평, 34평) |
| `호가(매/전)` | rich_text | 매매/전세 호가 |
| `평단가/갭` | rich_text | 평단가 및 갭 |
| `방문일` | date | 방문일 (YYYY-MM-DD) |
| `현장교통(5점)` | number | 1~5점 |
| `현장학군(5점)` | number | 1~5점 |
| `현장상권(5점)` | number | 1~5점 |
| `현장환경(5점)` | number | 1~5점 |
| `현장단지(5점)` | number | 1~5점 |
| `발품 요약` | rich_text | 한줄 발품 요약 |
| `임장 점수` | formula | **자동 계산** (입력 불필요) |

### 임장 점수 계산 공식
```
(현장교통 × 6) + (현장학군 × 4) + (현장상권 × 4) + (현장환경 × 3) + (현장단지 × 3)
```
만점: 100점 (5×6 + 5×4 + 5×4 + 5×3 + 5×3)

## 📄 페이지 본문(Content) 블록 구조

template.md 기반으로 노션 페이지 본문을 구성합니다:

1. **heading_1**: `🏢 [아파트명] 종합 입지 분석 보고서`
2. **callout** (📊, blue_background): 종합 점수 + 시세 정보 + 한줄 요약
3. **divider**
4. **heading_2**: `1. 사전 조사 (Static Data)`
   - **bulleted_list_item**: 교통 노드, 단지 스펙, 인프라, 학군 정보
5. **divider**
6. **heading_2**: `2. 현장 실사 (Dynamic Data)`
   - **heading_3**: `2.1 오감 및 분위기 체감`
     - **to_do**: 소음, 냄새, 경사, 관리, 보행, 단지 (checked/unchecked)
   - **heading_3**: `2.2 동/층별 실시간 일조량`
     - **table**: 동 번호 | 확인 층수 | 일조 상태 | 메모
7. **divider**
8. **heading_2**: `3. 임장 사진 아카이브`
   - **image** (external URL) 또는 안내 **callout**
9. **heading_2**: `추가 의견`
   - **paragraph**: 자유 텍스트
